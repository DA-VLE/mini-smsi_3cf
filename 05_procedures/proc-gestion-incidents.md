# Procédure de gestion des incidents de sécurité

## 1. Objectif

Définir les étapes à suivre lorsqu'un événement suspect ou un incident de sécurité est détecté.

## 2. Exemples d'incidents

- Email de phishing.
- Compte compromis.
- Ransomware.
- Perte d'ordinateur.
- Fuite de données.
- Site web modifié sans autorisation.
- Poste infecté par un malware.
- Indisponibilité du serveur de fichiers.

## 3. Détection et signalement

Tout utilisateur doit signaler rapidement un événement suspect au support IT ou au responsable sécurité.

Le signalement doit contenir si possible :
- date et heure ;
- utilisateur concerné ;
- poste ou application concernée ;
- description ;
- capture d'écran ;
- email suspect en pièce jointe si applicable.

## 4. Qualification

Le support IT qualifie l'événement :
- faux positif ;
- événement mineur ;
- incident confirmé ;
- incident critique.

Critères de criticité :
- nombre d'utilisateurs touchés ;
- présence de données sensibles ;
- impact sur la disponibilité ;
- suspicion de compromission ;
- propagation possible.

## 5. Réaction

### Cas phishing

1. Ne pas cliquer sur les liens.
2. Transférer l'email au support IT.
3. Supprimer l'email après analyse.
4. Si l'utilisateur a saisi son mot de passe : changer le mot de passe et vérifier les connexions.

### Cas ransomware

1. Isoler immédiatement le poste du réseau.
2. Ne pas éteindre si une analyse est nécessaire.
3. Prévenir le responsable IT.
4. Identifier les partages réseau touchés.
5. Vérifier les sauvegardes.
6. Lancer la restauration uniquement après validation.

### Cas compte compromis

1. Réinitialiser le mot de passe.
2. Révoquer les sessions actives.
3. Activer ou vérifier la MFA.
4. Contrôler les règles de transfert mail.
5. Analyser les connexions récentes.

## 6. Escalade

Un incident est escaladé à la direction si :
- le service est indisponible ;
- des données sensibles sont concernées ;
- plusieurs utilisateurs sont touchés ;
- un prestataire externe est impliqué ;
- une notification légale pourrait être nécessaire.

## 7. Clôture et retour d'expérience

Après traitement, un compte rendu est rédigé :
- cause probable ;
- impact ;
- actions menées ;
- preuves collectées ;
- actions correctives ;
- leçons apprises.

## 8. Preuves attendues

- Ticket incident.
- Captures d'écran.
- Journaux de connexion.
- Liste des actions réalisées.
- Rapport de clôture.
