# Guide de démarrage rapide

## Objectif

Ce dossier est un starter kit pour construire un projet GitHub crédible autour d'un mini-SMSI.

## Ordre de travail recommandé

1. Lire le README.
2. Adapter le nom de l'entreprise fictive si besoin.
3. Ouvrir l'inventaire des actifs et modifier les actifs selon le contexte choisi.
4. Ouvrir le registre des risques et vérifier les scénarios.
5. Ouvrir le plan d'actions et aligner chaque action avec un risque.
6. Relire les trois procédures.
7. Ajouter quelques captures d'écran ou preuves fictives si tu veux enrichir le repo.
8. Publier sur GitHub.

## Ce qu'il faut savoir expliquer en entretien

- Pourquoi tu as limité le périmètre.
- Pourquoi le phishing et le ransomware ressortent en risques critiques.
- Comment tu calcules le score Probabilité x Impact.
- Quelle différence existe entre risque brut et risque résiduel.
- Pourquoi une preuve attendue est importante dans un audit.
- Pourquoi les sauvegardes doivent être testées, pas seulement configurées.
- Pourquoi les droits doivent être revus régulièrement.
