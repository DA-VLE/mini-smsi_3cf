# Politique de sécurité de l'information

## 1. Objet

La présente politique définit les principes de sécurité applicables au système d'information de l'Aéroport Régional 3CF. Elle sert de document de référence pour le mini-SMSI du projet.

## 2. Périmètre

Le périmètre couvre :
- la messagerie professionnelle ;
- le serveur de fichiers ;
- les comptes utilisateurs et administrateurs ;
- les postes utilisateurs ;
- l'accès Internet et le pare-feu ;
- le site web public ;
- les sauvegardes ;
- les données administratives et RH.

Les systèmes de contrôle aérien, équipements industriels, vidéosurveillance et systèmes réglementaires spécialisés sont exclus du périmètre.

## 3. Objectifs de sécurité

Les objectifs de sécurité sont les suivants :

- **Confidentialité** : protéger les données internes, RH et administratives contre les accès non autorisés.
- **Intégrité** : éviter la modification non autorisée ou accidentelle des informations.
- **Disponibilité** : maintenir l'accès aux services essentiels, notamment messagerie, fichiers et Internet.
- **Traçabilité** : conserver des preuves suffisantes pour comprendre les actions importantes.
- **Amélioration continue** : suivre les incidents et actions correctives.

## 4. Principes directeurs

Les principes retenus sont :
- moindre privilège ;
- comptes nominatifs ;
- authentification multifacteur pour les accès sensibles ;
- sauvegardes régulières et testées ;
- sensibilisation régulière des utilisateurs ;
- traitement prioritaire des risques critiques ;
- conservation des preuves pour audit interne.

## 5. Rôles et responsabilités

| Rôle | Responsabilités |
|---|---|
| Direction | Valider la politique, arbitrer les moyens |
| Responsable sécurité / RSSI fictif | Suivre les risques, piloter le plan d'actions |
| Responsable IT | Mettre en place les mesures techniques |
| Managers | Valider les droits de leurs équipes |
| Utilisateurs | Respecter les règles et signaler les incidents |
| Prestataire IT | Appliquer les mesures demandées et fournir les preuves |

## 6. Exigences minimales

- Les comptes utilisateurs doivent être nominatifs.
- Les comptes administrateurs doivent être séparés des comptes bureautiques.
- Les accès aux dossiers sensibles doivent être validés par un responsable.
- Les sauvegardes doivent être testées au minimum chaque trimestre.
- Tout incident suspect doit être signalé immédiatement au support IT.
- Les postes doivent être verrouillés en cas d'absence.
- Les mises à jour critiques doivent être appliquées dans un délai raisonnable.

## 7. Revue de la politique

Cette politique est revue au moins une fois par an ou après un incident majeur.
